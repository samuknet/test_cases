The issue occurs when there is a text input near an image and the text input is 'filled up' with characters. As soon as the text input overflows a white rectangle clips the corner of the image. This happens as soon as the text field has a character entered that makes it's size too small to display all the characters.

The issue only occurs when the image has a CSS property set to anything other than 1. Also, the the image must be within a certain 'closeness' to the text input for the issue to occur.
